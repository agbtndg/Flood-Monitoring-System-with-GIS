## What is White-Box Testing?

**Simple Definition:**  
Testing with full knowledge of internal code structure, algorithms, and implementation.

**Our Implementation:**
- **Dynamic:** Execute code and validate results (25 tests)
- **Static:** Analyze code structure without execution (SonarQube tools)

**Why Both?**
- Dynamic finds runtime issues
- Static finds structural issues
- Together = comprehensive white-box coverage
